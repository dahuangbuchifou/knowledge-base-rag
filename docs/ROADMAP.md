## 🟡 P1 — 对话能力（核心体验）

### 1. 多轮对话上下文
- **问题：** 每次问答独立，无法追问
- **方案：** 
  - 前端存 `session_id`（百炼 API 已返回）
  - 后端维护会话状态（Redis 或内存）
  - 支持"上一个问题"、"继续刚才的话题"
- **预计耗时：** 1-2 小时
- **优先级：** 🔴 高

### 2. 回答引用溯源
- **问题：** 不知道答案来自哪个文档哪一页
- **方案：**
  - 百炼 API 开启 `has_thoughts: true` 获取引用
  - 前端展示引用来源（文档名 + 页码）
  - 点击可展开原文片段
- **预计耗时：** 1 小时
- **优先级：** 🔴 高

### 3. 流式输出（打字机效果）
- **问题：** 60 秒等待太枯燥
- **方案：**
  - 后端改 Server-Sent Events (SSE)
  - 前端逐字显示，像 ChatGPT 那样
- **预计耗时：** 1-2 小时
- **优先级：** 🟡 中

### 4. 富文本渲染
- **问题：** 表格、公式、代码显示效果差
- **方案：**
  - 集成 Markdown 渲染（marked.js）
  - 表格、代码高亮、数学公式（KaTeX）
- **预计耗时：** 1 小时
- **优先级：** 🟡 中

## 🟢 P2 — 扩展性（长远考虑）

### 5. 多知识库切换
- **问题：** 只能查发动机排放标准
- **方案：**
  - 前端下拉选择知识库
  - 后端配置多个 APP_ID
  - 支持动态添加知识库
- **预计耗时：** 2 小时
- **优先级：** 🟡 中

### 6. 用户体系
- **问题：** 无用户区分，无法做个性化
- **方案：**
  - 简易登录（手机号/邮箱）
  - 记录用户提问历史
  - 支持收藏常用问题
- **预计耗时：** 3-4 小时
- **优先级：** 🟢 低

### 7. 提问模板/快捷问题
- **问题：** 用户不知道能问什么
- **方案：**
  - 首页展示常见问题卡片
  - 支持一键填入示例问题
  - 搜索历史提问记录
- **预计耗时：** 1 小时
- **优先级：** 🟢 低
