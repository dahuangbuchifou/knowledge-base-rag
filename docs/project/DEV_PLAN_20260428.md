## 📅 三阶段执行路线

### 第一阶段：把"能用"变成"好用"（近期 MVP）

| 优先任务 | 为什么先做 | 预计耗时 |
|----------|------------|----------|
| 搜索优化（关键词 + 语义） | 用户最直接感受到"能不能找到答案" | 2-3 小时 |
| 答案引用来源 | 解决可信度问题，降低错误回答风险 | 1-2 小时 |
| 内容分类和模板 | 后续扩展、运营、权限管理都依赖清晰的内容结构 | 2-3 小时 |
| 用户反馈机制 | 让系统知道哪些问题没答好 | 1-2 小时 |
| 权限控制（已有 Basic Auth） | 初期就要避免敏感信息泄露 | 已完成 ✅ |

**MVP 目标：**
| 模块 | 近期应该做到的程度 |
|------|-------------------|
| 搜索 | 支持关键词 + 语义搜索 |
| 问答 | 基于知识库内容回答，并附带来源 |
| 权限 | 用户只能看到自己有权限的知识 |
| 内容管理 | 每篇文档有分类、标签、负责人、更新时间 |
| 反馈 | 用户可以标记答案是否有用 |
| 后台 | 管理员能看到高频问题、无结果问题、差评答案 |
| 安全 | 对敏感问题和越权问题进行拦截 |
| 扩展 | 预留数据源同步和 API 接口 |

## 🔴 P0 — 对话能力

### 1.1 多轮对话上下文管理
- **功能：** 支持追问、上下文记忆、会话历史
- **技术方案：** 
  - 百炼 API `session_id` 会话保持
  - 前端会话状态管理
- **预计耗时：** 2-3 小时
- **衡量指标：** 用户追问率、问答命中率

### 1.2 意图识别与路由
- **功能：** 识别用户意图，路由到不同处理模块
- **技术方案：**
  - 简单规则匹配（关键词）
  - 百炼 API 自带意图识别
- **预计耗时：** 1-2 小时

### 1.3 模糊问题澄清
- **功能：** 当用户问题模糊时，主动澄清
- **技术方案：**
  - 关键词匹配 + 置信度判断
  - 提供候选问题列表
- **预计耗时：** 1-2 小时

### 1.4 流式输出（SSE）
- **功能：** 打字机效果，提升等待体验
- **技术方案：** Server-Sent Events
- **预计耗时：** 1-2 小时

### 1.5 富文本渲染
- **功能：** 表格、公式、代码显示优化
- **技术方案：** marked.js + KaTeX
- **预计耗时：** 1 小时

## 💡 核心建议

**不要一上来做太复杂的 AI Agent。更实际的顺序是：**

1. **先把知识内容治理好** - 分类、标签、模板、负责人、更新时间
2. **再把搜索和问答做准** - 语义搜索、来源引用、多轮追问、低置信度提示
3. **同步补上安全能力** - 权限、审计、敏感信息保护、越权拦截
4. **最后再做平台化扩展** - API、插件、多系统接入、个性化推荐

**核心目标：** 把知识库从"文档展示网站"升级成 可搜索、可问答、可信任、可治理、可扩展的知识服务系统。
