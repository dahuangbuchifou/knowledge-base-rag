## 一、参考文件技术栈（本地数据库网页 - 参考_20260222.py）

### 核心技术
| 组件 | 技术选型 | 说明 |
|------|---------|------|
| **前端 UI** | Streamlit | 玻璃态设计、多主题切换 |
| **向量库** | ChromaDB | 本地向量存储 |
| **LLM 框架** | LlamaIndex | 检索增强生成 |
| **嵌入模型** | HuggingFace Embedding | 本地向量化 |
| **大模型** | Ollama | 本地部署 LLM |
| **检索增强** | RAG 融合（向量 + BM25） | 混合检索 |
| **重排序** | SBERT Rerank | 结果优化 |

### 核心功能
1. PDF 解析 → 向量化 → 建索引 → 对话检索回答
2. 支持"仅向量恢复 / 完整恢复（docstore/BM25）"两种模式
3. PDF 文本层体检、Chroma 写入校验
4. 安全增量更新（kb_manifest.json + 稳定 doc_id/file_hash）
5. 引擎失效控制与自愈、调试面板

### 部署方式
- **本地部署**：Streamlit 应用直接运行
- **数据本地化**：ChromaDB 本地存储
- **模型本地化**：Ollama 本地 LLM

## 四、分块脚本智能化方案分析

### 方案 A：本地运行（当前方案）
```bash
python3 scripts/split-docs.py
```

**优点：**
- ✅ 简单直接，依赖少（只需 PyMuPDF）
- ✅ 数据不出本地，隐私保护好
- ✅ 可自定义分块策略

**缺点：**
- ❌ 需要用户本地安装 Python 环境
- ❌ 需要手动安装依赖（pip install PyMuPDF）
- ❌ 用户操作门槛高

### 方案 B：GitHub Actions 自动化
```yaml
# .github/workflows/split-docs.yml
on:
  push:
    paths:
      - 'docs/standards/*.pdf'

jobs:
  split-docs:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      - uses: actions/setup-python@v4
        with:
          python-version: '3.9'
      - run: pip install PyMuPDF
      - run: python3 scripts/split-docs.py
      - run: git add -A && git commit -m "auto: 分块文档" && git push
```

**优点：**
- ✅ 用户无需本地操作
- ✅ 自动触发，上传 PDF 后自动分块
- ✅ 环境统一，避免依赖问题

**缺点：**
- ❌ 需要配置 GitHub Actions
- ❌ 需要授予写权限
- ❌ PDF 文件会暴露在 GitHub（公开仓库）

### 方案 C：百炼控制台直接上传 PDF
**优点：**
- ✅ 最简单，用户只需上传 PDF
- ✅ 百炼自动分块 + 向量化
- ✅ 无需额外脚本

**缺点：**
- ❌ 分块策略不可控
- ❌ 无法预览分块效果
- ❌ 无法版本管理分块文档

## 五、推荐方案

### 短期（本次项目）
**方案 C：百炼控制台直接上传 PDF**
- 理由：快速验证，减少开发成本
- 操作：上传 PDF → 百炼自动分块 → 创建应用

### 中期（优化体验）
**方案 B：GitHub Actions 自动化**
- 理由：平衡用户体验和数据控制
- 操作：上传 PDF → Actions 自动分块 → 推送分块文档 → 百炼上传

### 长期（生产环境）
**参考文件方案（本地部署）+ 百炼混合**
- 理由：数据隐私 + 云端弹性
- 操作：敏感数据本地处理，公开数据云端处理
