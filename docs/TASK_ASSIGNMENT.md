## 📝 各部门任务详情

### 工部（技术实现）

**任务 1：答案引用来源**
- 后端开启 `has_thoughts: true`
- 解析引用数据格式
- 前端展示引用卡片
- 低置信度答案标记

**任务 3：搜索优化（技术部分）**
- 后端添加 `/api/search` 接口
- 前端添加搜索框
- 搜索结果展示
- 搜索建议（可选）

**依赖：** 户部 API 能力调研结果

---

### 礼部（用户体验）

**任务 2：用户反馈机制**
- 前端添加反馈按钮（👍 有用 / 👎 无用 / 💡 建议）
- 后端添加 `/api/feedback` 接口
- 反馈数据保存到日志文件
- 反馈成功提示

**技术要求：**
- 前端：HTML/CSS/JS
- 后端：Express.js
- 存储：日志文件（feedback.log）

---

### 吏部（内容管理）

**任务 4：内容分类和模板**
- 设计知识分类体系
- 制定文档模板
- 整理现有文档
- 补充标签、负责人、更新时间

**分类体系（初稿）：**
```
发动机排放标准知识库
├── 国家标准
│   ├── GB17691-2018（重型车）
│   └── GB/T17692-2024（测试方法）
├── 测试标准
│   ├── WHSC 测试
│   ├── WHTC 测试
│   └── WHTC+WHSC 测试
├── 限值标准
│   ├── NOx 限值
│   ├── PM 限值
│   └── CO 限值
└── 常见问题
    ├── 测试流程
    ├── 公式计算
    └── 标准解读
```

**文档模板：**
```markdown
# [文档标题]

## 基本信息
- **分类：** [一级分类] / [二级分类]
- **标签：** [标签 1, 标签 2, 标签 3]
- **负责人：** [姓名]
- **更新时间：** [YYYY-MM-DD]
- **版本：** [v1.0]

## 内容
[文档内容]

## 参考资料
- [参考 1]
- [参考 2]
```

### 户部（信息调研）

**任务 5：API 能力调研**
- 调研百炼 API 文档
- 确认是否支持搜索功能
- 确认是否返回引用数据
- 确认引用数据格式
- 确认是否支持多轮对话（session_id）
- 输出调研报告

**调研清单：**
- [ ] 搜索 API 接口
- [ ] 引用数据格式
- [ ] 多轮对话支持
- [ ] 限流策略
- [ ] 错误码说明

---

### 兵部（安全评估）

**任务 6：安全评估**
- 反馈机制安全审查
- 防止恶意反馈（刷赞/刷踩）
- 反馈数据匿名化
- 敏感信息保护
- 输出安全建议

**评估要点：**
- [ ] 反馈频率限制
- [ ] IP 白名单/黑名单
- [ ] 敏感词过滤
- [ ] 数据脱敏

---

### 中书省（规划审核）

**任务 7：整体规划审核**
- 审核各部门任务分配
- 协调任务依赖关系
- 监控进度
- 输出进度报告

---

### 门下省（质量审核）

**任务 8：质量审核**
- 审核代码质量
- 审核文档规范
- 测试验收
- 输出审核报告
